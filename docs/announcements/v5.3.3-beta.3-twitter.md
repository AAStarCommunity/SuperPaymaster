```
🛡️ SuperPaymaster v5.3.3-beta.3 is live on Sepolia — a security hotfix on top of beta.2.

The 2026-06-11 comprehensive audit surfaced 2 more High severity issues. Both are now fixed, Codex-reviewed, and deployed.
```

**2/ H-1 — credit ceiling in debt-fallback**
```
🟠 H-1: When postOp can't burn xPNTs (user drained balance mid-UserOp), it falls back to debt recording. Before beta.3, no credit limit check — attackers could accumulate unlimited operator debt.

Fix: ceiling enforced in fallback. Over-ceiling → auto-blocked (DVT/BLS unlock only, not owner).
```

**3/ H-2 — xPNTs emergency stop**
```
🟠 H-2: xPNTsToken's emergencyDisabled flag lets communities halt all token movement. But transferFrom's auto-approved path (`to == msg.sender`) skipped it entirely.

Fix: self-pull path now goes through the full emergency + daily rate-limit check.
```
