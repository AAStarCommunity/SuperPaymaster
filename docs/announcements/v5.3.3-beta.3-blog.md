# SuperPaymaster v5.3.3-beta.3 — Security Hotfix Release (Sepolia)

> **TL;DR**: v5.3.3-beta.3 is a security-only hotfix release. Two additional High severity findings
> from the comprehensive 2026-06-11 audit have been patched, Codex-reviewed, and deployed on
> Sepolia. All proxy addresses are unchanged. Integrators do not need to update contracts;
> SDK config needs one address update (xPNTsFactory).

---

## What changed from beta.2

This release adds **2 High severity fixes** on top of beta.2's 6 critical/high fixes. No new
features. No breaking changes to the SP or Registry proxy interface.

| ID | Severity | Fix |
|----|----------|-----|
| H-1 | High | **Credit ceiling bypass** — `_recordDebt()` now enforces the user's credit limit in the debt-fallback path. Over-ceiling debt is isolated to `pendingDebts` and the user is auto-blocked (unlockable only via DVT/BLS consensus). |
| H-2 | High | **transferFrom emergency switch bypass** — `xPNTsToken.transferFrom()` `autoApproved` branch now checks `emergencyDisabled` and the daily rate-limit, closing a path that let users bypass the community emergency stop. |

**Full audit report**: [`docs/audit/comprehensive-audit-2026-06-11.md`](https://github.com/AAStarCommunity/SuperPaymaster/blob/main/docs/audit/comprehensive-audit-2026-06-11.md)

---

## What stays the same

Everything the beta.2 integration guide describes is still valid:

- All SuperPaymaster and Registry **proxy addresses are identical** — no SDK contract config update needed for the hot path.
- All 8 capabilities from beta.2 remain live (gasless sponsorship, dual-channel human+agent, reputation credit/debt, MicroPaymentChannel, x402 settlement, etc.).
- Test prerequisites (SBT eligibility, `dryRunValidation`, `paymasterAndData` format) are unchanged.

---

## ⚠️ One config update needed (community operators only)

`xPNTsFactory` was redeployed to carry the H-2 fix (EIP-1167 clones cannot be upgraded in-place).

| | Old | New |
|---|---|---|
| xPNTsFactory | `0xC4f5A121c426734CC1c0DbE57f6A2Dd764E278e4` | `0xc312CAFcb49dFe3aB76bFB2F3e37CaEdBa65ccd9` |

- **Existing operators**: their xPNTs token and all balance/config in SP is preserved. Operators
  who had already configured before beta.3 were migrated automatically (Anni demo operator done
  on-chain). Any operator needing a fresh token deploy should call the new factory.
- **SDK integrators using `deployxPNTsToken`**: update `config.sepolia.json` → `xPNTsFactory` to
  the new address above.
- **Gasless UserOp flow**: no change needed — SP.xpntsFactory() already points to the new factory.
