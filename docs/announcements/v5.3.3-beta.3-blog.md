## 什么是 SuperPaymaster？

SuperPaymaster 是面向 ERC-4337 账户抽象的**去中心化 Gas 支付基础设施**。

用户与 dApp 交互时，Gas 费用是最大的使用摩擦——用户必须持有 ETH，并手动管理 Gas 余额。SuperPaymaster 彻底消除了这个摩擦：**社区用自己的代币（xPNTs）为用户代付 Gas**，用户看不到 ETH，也不需要 ETH。

这不是一个 Paymaster 合约，而是一套**可组合的 Gas 赞助基础设施**，由社区注册、社区运营、社区治理。

---

## 核心特性

### 1. 无 Gas 用户体验

用户发送 ERC-4337 UserOperation，Gas 费由社区 operator 的 xPNTs 池子代缴。用户无需感知 ETH 的存在。链上结算完全透明，无需信任任何中间方。

### 2. 两种部署模式

| 模式 | 合约 | 适用场景 |
|------|------|---------|
| **AOA+ (SuperPaymaster)** | 共享多社区 Paymaster | 多个社区共用一套基础设施，Registry 统一管理注册、质押、声誉 |
| **AOA (PaymasterV4)** | 每社区独立 Paymaster | 通过 EIP-1167 最小代理工厂按社区部署，完全自主运营 |

两种模式使用同一套 Registry，operator 配置、xPNTs 代币、声誉数据互通。

### 3. xPNTs 社区 Gas 代币

每个社区通过 `xPNTsFactory` 部署自己的 xPNTs 代币（EIP-1167 克隆）。用户在社区场景中积累 xPNTs，用于抵扣 Gas。

xPNTs 的关键属性：
- 社区可设置每日兑换上限（`dailyRateLimit`）
- 社区可一键触发紧急停机（`emergencyDisabled`）
- SP 的 `autoApproved` 快速结算路径受紧急开关保护（beta.3 H-2 修复）

### 4. 信用 / 债务系统

用户的 xPNTs 余额不足时，SP 可临时垫付并记录债务（`debt`），后续用户补充余额后自动偿还。

- 债务上限受信用额度控制（`creditLimit`），beta.3 H-1 修复了 fallback 路径的上限绕过漏洞
- 超额债务用户自动进入封锁状态，只能通过 DVT/BLS 共识网络解除（不受合约 owner 控制）

### 5. 双通道 Gas 赞助（人类 + AI Agent）

SP 通过两个身份通道判定是否赞助：
- **SBT 持有者（人类用户）**：MySBT 灵魂绑定代币持有者
- **注册 Agent（AI 智能体）**：通过 ERC-8004 AgentIdentityRegistry 注册的自主 AI 代理

每个 operator 可为 Agent 单独配置赞助策略（`agentPolicies`）：按 BPS 费率 + 每日 USD 上限分级控制。

### 6. x402 微支付结算

SP 支持 x402 协议（HTTP-native payment）的链上结算：
- `settleX402Payment()`：通过 EIP-3009 `transferWithAuthorization` 结算 USDC（节省 19% gas）
- `settleX402PaymentDirect()`：通过 `transferFrom` 结算 xPNTs（factory autoApproved 路径）

支持 Facilitator 白名单，可接入 Coinbase 托管 Facilitator 或自托管版本。

### 7. MicroPaymentChannel

链下快速支付通道合约（`MicroPaymentChannel`），支持 AI Agent 和 Web 服务之间的微支付流。与 x402 配合使用，降低链上结算频率。

### 8. DVT / BLS 安全层

- `BLSValidator`：BLS12-381 签名聚合验证
- `DVTValidator`：分布式验证节点集合管理
- `BLSAggregator`：聚合多个 DVT 节点签名，执行链上罚没（slashing）

两层罚没机制：
- Tier 1：SP 罚没 operator 的 aPNTs（运营资金）
- Tier 2：GTokenStaking 罚没 operator 的 GToken 质押（治理资金）

## AAStar 技术栈

SuperPaymaster 是 AAStar 账户抽象技术栈的 Gas 结算层：

```
用户 / AI Agent
      ↓
AirAccount (账户层)   — passkey、session key、recovery、agent
      ↓
SuperPaymaster (Gas 层) — xPNTs 赞助、credit、x402、DVT
      ↓
ERC-4337 EntryPoint
      ↓
Ethereum / OP Mainnet
```

配套 SDK：`aastar-sdk` 提供 `x402Actions`、`agentActions`、`channelActions` 三组 L1 操作模块。

## 如何开始

**社区 Operator（部署自己的 xPNTs + 加入 SP）：**

```bash
# 1. 克隆仓库
git clone https://github.com/AAStarCommunity/SuperPaymaster

# 2. 部署到 Sepolia（需要 .env.sepolia）
./deploy-core sepolia

# 3. 初始化测试账户
./prepare-test sepolia
```

**开发者（集成 SDK）：**

参考 [`aastar-sdk`](https://github.com/AAStar/aastar-sdk) 的 `x402Actions` 和 gasless UserOp 示例。
