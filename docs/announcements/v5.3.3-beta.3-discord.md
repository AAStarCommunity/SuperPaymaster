## 📢 Channel announcement

```
🎉 SuperPaymaster v5.3.3-beta.3 is live on Sepolia — and this is our first public release.

SuperPaymaster is decentralized gas sponsorship infrastructure for ERC-4337 Account Abstraction.
Communities deploy their own xPNTs tokens and use them to pay gas on behalf of users — no ETH required.

🔑 Core capabilities (all live on Sepolia):

⛽ Gasless UX
  Users send UserOperations without holding ETH. Gas is covered by the community's xPNTs token pool.

🏘️ Two deployment modes
  • AOA+ (SuperPaymaster): shared multi-community infrastructure, Registry-managed stake + reputation
  • AOA (PaymasterV4): independent per-community paymaster via EIP-1167 minimal proxy

🤖 Human + AI Agent dual-channel sponsorship
  SBT holders (humans) and ERC-8004 registered AI agents both qualify for gas sponsorship.
  Operators can configure tiered Agent sponsorship policies (BPS rate + daily USD cap).

💳 Credit / debt system
  Users with insufficient xPNTs can accumulate debt up to their credit limit.
  Over-limit users are auto-blocked — unlockable only via DVT/BLS consensus.

⚡ x402 micropayment settlement
  On-chain settlement for HTTP-native x402 payments.
  EIP-3009 USDC path (19% gas savings) + xPNTs direct path.

🔒 DVT / BLS security layer
  Two-tier slashing: aPNTs (operational) + GToken stake (governance).
  Slash execution requires BLS aggregated signatures from DVT validator network.

🛡️ Audit-hardened
  8 fixes (2 Critical + 6 High) across beta.2 + beta.3, all Codex-reviewed.

📍 Sepolia testnet only — not for mainnet value.
📦 https://github.com/AAStarCommunity/SuperPaymaster/releases/tag/v5.3.3-beta.3
```

## 🧵 Thread: 安全修复说明（beta.3 新增）

```
beta.3 修复了综合安全审计发现的 2 个 High 级别漏洞：

🟠 H-1 — Credit ceiling bypass（debt-fallback 路径）
  场景：用户在 UserOp 执行中途清空 xPNTs，postOp 无法正常销毁代币，
  触发 debt fallback。旧版本 fallback 路径未检查信用额度，
  攻击者可无限积累 operator 债务。
  修复：fallback 路径现在强制检查 existingDebt + pendingDebts + bill ≤ creditLimit。
  超限用户自动封锁，仅限 DVT/BLS 网络解除，不受 owner 控制。

🟠 H-2 — xPNTs emergency switch bypass（transferFrom autoApproved 路径）
  场景：xPNTsToken 有 emergencyDisabled 标志，社区可一键停止所有代币流动。
  但 transferFrom 中 to == msg.sender（自拉取）的快速路径跳过了该检查。
  攻击者可在紧急停机期间继续转出代币。
  修复：自拉取路径（to == msg.sender && to != SUPERPAYMASTER）现在
  强制走完整的 emergencyDisabled + dailyRateLimit 检查。

完整审计报告：
https://github.com/AAStarCommunity/SuperPaymaster/blob/main/docs/audit/comprehensive-audit-2026-06-11.md
```
