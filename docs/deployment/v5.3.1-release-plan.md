## 二、部署核心步骤

### 阶段 0：分支清理（本文档 A+B 分支处理）

```
1. merge chore/deploy-anvil-verify → main
2. merge ci/cla-allowlist-add-owners → main
3. merge fix/p2-reputation-rule-bounds → main (需 review)
4. merge fix/xpnts-factory-sp-propagation → main (需 review)
5. 关闭 C 中已被覆盖的分支
```

### 阶段 1：本地 Anvil 验证

```bash
# 1. 强制重部署所有合约到本地 anvil
./deploy-core anvil --force

# 2. 跑完整回归（7 个 Check 脚本 + E2E 基础路径）
./run_full_regression.sh

# 3. 准备测试账户
./prepare-test anvil

# 4. E2E gasless 测试
cd script/gasless-tests && ./run-all-tests.sh
```

**通过标准**: 所有 Check01–Check08 + VerifyV3_1_1 均绿，gasless E2E 无报错。

### 阶段 2：Sepolia 部署

```bash
# 1. 部署核心合约（UUPS 升级 Registry + SuperPaymaster impl）
./deploy-core sepolia --force

# 2. 链上版本确认
./version-check-onchain.sh

# 3. 初始化测试账户
./prepare-test sepolia

# 4. 运行 E2E 测试全套
cd script/gasless-tests && ./run-all-tests.sh

# 5. x402 facilitator 测试
node test-x402-facilitator.js
node test-micropayment-channel.js
```

**通过标准**: 
- 链上 `version()` 返回与本地合约版本一致
- 所有 E2E 测试组 A~G 通过
- x402/micropayment 场景无报错

### 阶段 3：主网（Wave 2 完成后）

> **前提**: P0-12a + P0-12b 必须完成并通过 Sepolia 验证  
> 届时单独开新的发布计划文档

## 三、回滚策略

| 场景 | 回滚方法 |
|------|---------|
| Anvil 部署失败 | 本地重跑 `--force`，无影响 |
